# SystemsThinking

**Structural analysis that finds the causes underneath the symptoms.**

The Iceberg Model: most interventions happen at the event level (what happened) because that's what's visible. Systems thinking drops below the waterline to find patterns over time, underlying structures, and the mental models that generated those structures. Interventions at the structural level are durable. Interventions at the event level are repeated.

## The Iceberg Model

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
     EVENTS          [visible, reactive]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    PATTERNS         [trends over time]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   STRUCTURES        [what enables the pattern]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  MENTAL MODELS      [the assumptions generating the structure]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Meadows' Leverage Points (highest to lowest impact)

1. The power to change the paradigm (mental model)
2. The goals of the system
3. The rules of the system
4. The structure of information flows
5. The gain around positive feedback loops
6. The strength of negative feedback loops
7. Delays in the system
8. The size of stocks and flows

Most interventions target the bottom of this list. Durable change requires working higher.

## Common System Archetypes

- **Fixes that fail:** quick fix relieves symptoms, creates side effect that worsens the problem
- **Shifting the burden:** symptomatic solution weakens the capacity to solve the fundamental problem
- **Limits to growth:** growth collides with an unseen constraint; pushing harder worsens it
- **Escalation:** two actors respond to each other's actions, both escalate

```
MENTAL MODEL (root): [the assumption that generated the structure]

ARCHETYPE MATCH: [if applicable]

```

## When to Use

- Recurring problems that survive surface fixes
- Unintended consequences after system changes
- Stakeholder disagreements about root cause (they're often at different iceberg levels)
- "Why does this keep happening?"
